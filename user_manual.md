# คู่มือการใช้งานระบบบัญชีการเงินร้านกาแฟ Jamjuree Cafe
### โรงเรียนวังน้ำเย็นวิทยาคม สำนักงานเขตพื้นที่การศึกษามัธยมศึกษาสระแก้ว

ระบบงานบัญชีร้านกาแฟได้รับการออกแบบมาให้ใช้งานได้สะดวกรวดเร็ว ปลอดภัย และป้องกันความผิดพลาด โดยแบ่งบัญชีผู้ใช้งานออกเป็น **3 บทบาทหน้าที่หลัก** ตามสิทธิ์การเข้าถึงข้อมูล ดังนี้:

---

## ☕ บทบาทที่ 1: เจ้าหน้าที่คาเฟ่ (Staff)
*เหมาะสำหรับ: นักเรียนผู้ทำหน้าที่ขายของหน้าร้าน หรือคุณครูประจำเวรประจำวัน*
*รหัสผ่านตั้งต้น: `cafe2026`*

### 1. การเข้าสู่ระบบ
1. เปิดหน้าเว็บไซต์หลัก กรอกรหัสผ่าน `cafe2026` ในช่องกรอกรหัสผ่านผู้ใช้งาน
2. กดปุ่ม **"เข้าสู่ระบบ"** ระบบจะนำทางไปยังเมนู **"บันทึกบัญชีรายวัน"** โดยอัตโนมัติ

### 2. การบันทึกธุรกรรมรายวัน (รายรับ - รายจ่าย)
1. ในหน้า **"บันทึกบัญชีรายวัน"** กดปุ่มสีน้ำตาล **"➕ เพิ่มรายการใหม่"** ด้านขวาบน
2. **หน้าต่างแบบฟอร์ม** จะแสดงขึ้นมา:
   *   **ประเภทธุรกรรม**: กดเลือกระหว่าง **"รายรับ"** (เช่น ยอดขายประจำวัน) หรือ **"รายจ่าย"** (เช่น ซื้อนมสด)
   *   **วันที่ทำรายการ**: ระบบจะใส่วันที่ปัจจุบันให้โดยอัตโนมัติ (สามารถกดแก้ไขย้อนหลังได้)
   *   **รายการ / คำอธิบาย**: พิมพ์รายละเอียดธุรกรรมลงในช่องพิมพ์ หรือ **กดเลือกรายการยอดนิยมอัจฉริยะ (Suggestions)** ที่แสดงขึ้นมาด้านล่างเพื่อป้องกันการสะกดพิมพ์ผิดพลาด
   *   **จำนวนเงิน**: กรอกตัวเลขเฉพาะค่าที่เป็นบวก (ระบบห้ามใส่เครื่องหมายติดลบ `-` ป้องกันการสับสน)
   *   **ผู้ทำบันทึก**: กรอกชื่อผู้ทำรายการจริงในวันนั้น
   *   **หมายเหตุ**: กรอกรายละเอียดเพิ่มเติม เช่น เลขใบเสร็จ หรือข้อความอื่นๆ (ถ้ามี)
3. กดปุ่ม **"บันทึกข้อมูล"** 
   *   *ในระหว่างประมวลผลเซฟลงกูเกิลชีตคลาวด์ ปุ่มและช่องกรอกจะถูกล็อกป้องกันการกดซ้ำ และจะแสดงคำว่า **"กำลังบันทึก..."** เมื่อเสร็จสิ้น หน้าต่างจะปิดลงอัตโนมัติ*

### 3. การค้นหาและตรวจสอบรายการบัญชี
*   คุณครูหรือเจ้าหน้าที่สามารถพิมพ์คำค้นหาในช่อง **"🔍 ค้นหารายการบัญชี..."** เพื่อกรองสืบค้นเฉพาะรายการที่สนใจได้อย่างรวดเร็ว

### 4. การออกใบรายงานบัญชี PDF
1. กดเลือกเมนูฝั่งซ้าย **"จัดทำรายงาน PDF"**
2. เลือกประเภทช่วงเวลาที่ต้องการออกเอกสาร (รายสัปดาห์, รายเดือน, รายไตรมาส, หรือรายปี)
3. ระบบจะวาดหน้ากระดาษใบรายงานที่มีโลโก้โรงเรียนและลายน้ำอย่างเป็นทางการ
4. กดปุ่ม **"พิมพ์รายงาน (PDF)"** ด้านบน บราวเซอร์จะเปิดหน้าต่างสั่งพิมพ์กระดาษ A4 เพื่อบันทึกเป็นไฟล์ PDF ลงในเครื่องคอมพิวเตอร์หรือพิมพ์ออกเครื่องปริ้นท์ได้ทันที

---

## 📊 บทบาทที่ 2: ผู้ดูแลระบบ (Admin)
*เหมาะสำหรับ: คุณครูหัวหน้าโครงการ, คุณครูฝ่ายการเงิน หรือผู้บริหารโรงเรียน*
*รหัสผ่านตั้งต้น: `admin@2026`*

### 1. การเข้าสู่ระบบ
1. กรอกรหัสผ่านแอดมิน `admin@2026` ที่หน้าแรกเพื่อเข้าสู่ระบบ
2. แอดมินจะได้รับสิทธิ์การเห็นเมนู **"หน้าแรก (แดชบอร์ด)"** และ **"ตั้งค่าสิทธิ์ & API"** เพิ่มขึ้นมาทันที

### 2. การวิเคราะห์ข้อมูลผ่านแดชบอร์ด (Dashboard)
1. ในหน้า **"หน้าแรก (แดชบอร์ด)"** แอดมินสามารถดูสรุปตัวเลขความเคลื่อนไหวการเงินจริงได้ผ่านการ์ดสรุป 3 ใบหลัก:
   *   **ยอดรวมรายรับจริง**: สรุปยอดขายจริงทั้งหมดโดยไม่มีการรวมยอดยกมาจากปีก่อนมาปะปน
   *   **ยอดรวมรายจ่ายจริง**: สรุปรายจ่ายหมุนเวียนจริงของร้าน
   *   **ยอดเงินคงเหลือสุทธิ**: ยอดสะสมเงินคงคลังปลายทางจริง ณ ปัจจุบัน
2. **กราฟวิเคราะห์ย้อนหลัง 6 เดือน**: แสดงความต่างของรายรับและรายจ่ายแบบเปรียบเทียบในรูปกราฟแท่งประยุกต์สวยงาม
3. **สัดส่วนค่าใช้จ่ายหลัก**: แสดงแผนภูมิจำแนกสินค้าที่ซื้อมูลค่าสูงสุด 5 อันดับแรกในรูปแบบร้อยละเพื่อวิเคราะห์งบประมาณ
4. **พิมพ์ตารางสรุปรายเดือนสะสม**: ที่ตารางสรุปด้านล่างสุด แอดมินสามารถกดปุ่ม **"พิมพ์รายงาน PDF"** เพื่อสั่งพิมพ์ตารางสรุปงบกระแสเงินสะสมประวัติร้านในรูปแบบ A4 ได้ในคลิกเดียว

### 3. การควบคุมจัดการแก้ไข/ลบธุรกรรม
*   แอดมินมีสิทธิ์กดปุ่ม **ไอคอนรูปดินสอ ✏️** เพื่อแก้ไขรายการข้อมูลที่บันทึกผิดพลาด หรือกดปุ่ม **ไอคอนถังขยะ 🗑️** เพื่อลบธุรกรรมนั้นออกจาก Google Sheets โดยตัวระบบจะมีหน้าต่าง Confirm Modal แสดงเตือนความปลอดภัยทุกครั้งก่อนดำเนินการลบจริง

### 4. การจัดการล้างข้อมูลเก่าเพื่อเริ่มต้นใหม่ (Reset Database)
*   หากแอดมินต้องการล้างข้อมูลเพื่อเริ่มภาคเรียนใหม่ สามารถเข้าไปที่เมนู **"ตั้งค่าสิทธิ์ & API"** และกดปุ่มลบธุรกรรมเพื่อทำความสะอาดฐานข้อมูลกูเกิลชีตทุกเดือนได้ทันที

---

## ⚙️ บทบาทที่ 3: แอดมินระบบเชื่อมต่อ API / ผู้พัฒนาระบบ
*เหมาะสำหรับ: คุณครูคอมพิวเตอร์, เจ้าหน้าที่ไอทีผู้รับผิดชอบดูแลสปรีดชีต หรือนักพัฒนา*
*รหัสผ่านตั้งต้น: `admin@api2026`*

### 1. การปลดล็อคเข้าสู่เมนูตั้งค่าลึก
1. สลับไปที่เมนู **"ตั้งค่าสิทธิ์ & API"** ฝั่งซ้าย
2. กรอกรหัสผ่านผู้พัฒนาสำหรับเข้าใช้งานลึกคือ `admin@api2026`
3. *เพื่อความปลอดภัย ระบบจะทำการล็อกหน้าจอ API นี้โดยอัตโนมัติหากผู้ใช้งานกดสลับแท็บไปดูเมนูหน้าอื่น*

### 2. การควบคุมฐานข้อมูลและสลับโหมดออนไลน์
*   **การเชื่อมต่อคลาวด์**: สามารถก๊อปปี้ลิงก์ URL สคริปต์ Google Apps Script Web App ของโรงเรียนมาวางในช่องและกดบันทึกเพื่อลิงก์ไปยังไฟล์ Google Sheets หลัก
*   **สลับโหมด Online / Offline**: 
    *   **Online**: เชื่อมต่อเขียนข้อมูลไปยัง Google Sheets ปลายทาง
    *   **Offline**: ใช้ข้อมูลจำลองบันทึกในเครื่อง สำหรับใช้ในการซักซ้อมหรือจำลองสอนนักเรียนในวิชาเรียนโดยไม่มีผลต่อข้อมูลจริงบนสเปรดชีตหลักของร้านกาแฟ

### 3. การเปลี่ยนรหัสผ่านใช้งานระบบ (Passwords Management)
*   ระบบรหัสผ่านทั้ง 3 บทบาทถูกจัดเก็บไว้บนกูเกิลชีตแท็บ `Passwords` แอดมินระบบสามารถเปิด Google Sheets หลักและพิมพ์แก้ไขรหัสผ่านในคอลัมน์ B ได้โดยตรง ซึ่งระบบของตัวเว็บจะทำหน้าที่ดึงค่ารหัสผ่านตัวอัปเดตใหม่มาตรวจสอบสิทธิ์แบบเรียลไทม์ทันทีโดยไม่ต้องรื้อแก้โค้ดคอมพิวเตอร์
